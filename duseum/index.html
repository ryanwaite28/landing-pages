What is Duseum

A museum metaphor
with real mechanics.

Duseum operates on a dual-profile model — every account holds a Viewer profile for discovery and an optional Author profile for publishing. Access is layered: free browsing, platform-wide subscriptions, and direct per-author subscriptions that unlock private collections.

A single account can be both creator and collector simultaneously. Authors earn through subscriber revenue and platform feature placements. Viewers get a curated, museum-quality discovery experience.

Platform Features

Everything a real
art platform needs.

Built with production-grade depth across discovery, monetization, social engagement, and creator tools.

01

Layered Access Tiers

Free browsing, platform subscriptions for unlimited public art, and per-author subscriptions for private collections. Access enforced server-side on every request.

02

Daily & Weekly Featured Authors

A random author spotlighted every day at no cost. Authors book paid weekly feature slots (up to 10 per week, first-come first-served, one booking per 3-month window).

03

Follow & Email Notifications

Viewers follow authors and receive SES email notifications on new uploads. Preferences are per-author and globally configurable, with CAN-SPAM compliant one-click unsubscribe.

04

Private Collections

Authors designate a private section accessible only to paying subscribers. Private pieces served via CloudFront signed URLs with 1-hour TTL — never directly exposed.

05

Stripe-Powered Monetization

Platform and author subscriptions via Stripe Billing. Weekly feature fees via Stripe Payment Intents. Webhook processing is idempotent via a DynamoDB deduplication table.

06

Collections & Tags

Authors curate named collections mixing public and private pieces. Viewers see only pieces their tier grants access to, with an honest "N pieces visible to you" count.

07

Social Interactions

Reactions (LOVE, WOW, FIRE, INSPIRED), threaded comments, pinned comments, and author replies. Authors control comment availability per piece.

08

Author Analytics

Dashboard surfaces view counts, follower and subscriber counts, MRR, notification delivery summaries, and upcoming feature booking history.

09

Admin Controls

Full admin panel for moderation, daily feature overrides, booking cancellations with Stripe refunds, and live platform config updates — no deploys needed.

Technology

Fully serverless.
Zero idle compute.

Every Lambda scales independently. DynamoDB on-demand billing. CloudFront for media. Infrastructure as code with AWS CDK TypeScript. Built to cost nearly nothing at zero traffic — scales linearly with growth.

AWS Lambda API Gateway HTTP API DynamoDB (single-table) Amazon Cognito CloudFront + S3 AWS SES SQS + DLQ EventBridge AWS CDK (TypeScript) React 18 + Vite TypeScript 5 Middy (middleware) Stripe Billing Stripe Connect Tailwind CSS Zustand React Query esbuild Turborepo GitHub Actions + OIDC MiniStack (local dev) Vitest + Playwright
// Lambda inventory — one domain per function, scales independently
artworks-lambda → CRUD, access tier enforcement, SQS publish on publish
users-lambda → profiles, follow/unfollow, notification preferences
subscriptions-lambda → Stripe Checkout session creation, billing portal
sub-webhook-lambda → SQS trigger — Stripe events, idempotent via DynamoDB
notifications-lambda → SQS trigger — SES fan-out, preference enforcement
features-lambda → daily spotlight, weekly bookings, availability calendar
maintenance-lambda → EventBridge daily + Monday — rotation, cleanup jobs
media-lambda → S3 presigned URLs, upload intent lifecycle
